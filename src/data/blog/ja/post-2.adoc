== はじめに

この投稿では、AstroプロジェクトにTailwind CSS v4を導入し、効率的にモダンなデザインを構築するための手順と考慮事項を説明します。Tailwind CSS v4は、これまでのバージョンから大幅な改善が加えられており、より高速な開発体験と優れたパフォーマンスを提供します。

== Tailwind CSS v4の導入

AstroプロジェクトでTailwind CSS v4を使用するには、 `@tailwindcss/vite` プラグインを利用するのが最も簡単な方法です。 `astro.config.mjs` に以下の設定を追加することで、Viteを通じてTailwind CSS v4の機能を活用できます。

.astro.config.mjs
[source,js]
----
import { defineConfig } from "astro/config";
import tailwindcss from "@tailwindcss/vite";

export default defineConfig({
    vite: {
        plugins: [tailwindcss()],
    },
});
----

この設定により、 `@tailwind` ディレクティブをCSSファイルで使用できるようになり、ユーティリティファーストのアプローチでスタイリングを進めることができます。

== Typographyプラグインの活用

ブログ記事のようなコンテンツが中心のサイトでは、適切なタイポグラフィの適用が重要です。 `@tailwindcss/typography` プラグインを導入することで、 `prose` クラスをHTML要素に適用するだけで、読みやすいデフォルトスタイルを自動的に適用できます。

まず、 `pnpm add -D @tailwindcss/typography` でプラグインをインストールし、 `global.css` などのCSSファイルで `@plugin "@tailwindcss/typography";` をインポートします。

.global.css
[source,css]
----
@import "tailwindcss";
@plugin "@tailwindcss/typography";

/* その他のグローバルスタイル */
----

そして、記事のコンテンツを囲む要素に `prose` クラスを付与するだけで、見出し、段落、リストなどの要素に美しいタイポグラフィが適用されます。

== まとめ

AstroとTailwind CSS v4の組み合わせは、パフォーマンスと開発体験の両面で非常に強力な選択肢となります。特に、コンテンツ駆動型のウェブサイトを構築する際には、両者の利点を最大限に活かすことができるでしょう。
